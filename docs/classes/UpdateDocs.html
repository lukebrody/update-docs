Class UpdateDocs

Implements syncing test cases to code examples in documentation files.

  1. Create a new UpdateDocs instance. This automatically reads files, but you can use read() to re-read them.

  2. Use collectExamples to scan through test files and collect code between // start docs Example Name and // end docs comments. Multiple blocks for the same example will be concatenated.

  3. Use replaceExamples to scan through documentation files and update code blocks under comments.

  4. You can use updateExamples to do both of the above at once.

  5. Run the tests. Use calls of replaceToken to inject test data into the generated examples. For example, you might want to have a comment about the value of an expression.

  6. Use write to write out documentation changes to the file system.
